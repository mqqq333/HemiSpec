    SCH API validation tutorial¶

    This tutorial records a reproducible SCH validation workflow for introducing the HemiSpec Python API and the equivalent CLI workflow. It includes a small smoke-test recipe for demonstrations and an internal full-cohort validation summary (N=948) for reporting software agreement and downstream hemisphere-classification accuracy. Replace the paths below with your cohort location before sharing outside the internal cluster.

    Private-data boundary

    The SCH cohort and manual ANS/RNS outputs are private/internal assets. Do not publish subject-level paths, NIfTI files, or derived private tables unless the data owner has approved release. The command examples below use a three-subject smoke test for reproducibility; the full-cohort section reports aggregate validation numbers only.

    Goal¶

    Run HemiSpec on preprocessed gray-matter maps, extract voxelwise and Glasser ROI ANS/RNS results, and compare the new API outputs against a previously generated manual result directory.

    The validation answers four questions:

    1. Can the PyPI-installed package import and find the DGN model bundles?
    2. Can the Python API run the bilateral DGN -> ANS/RNS workflow on real SCH-style inputs?
    3. What files are produced, and where should users look for voxel maps and ROI features?
    4. How close are the new API outputs to the existing manual ANSRNS_bi_stable outputs?
    5. What downstream hemisphere-classification accuracy is obtained from the extracted ROI features?

    Step 1: create a small input subset¶

    Use symlinks so the smoke test does not copy private NIfTI payloads:

    mkdir -p "$OUT/input_subset" "$OUT/logs"
rm -f "$OUT/input_subset"/*.nii.gz
python3 - <<'PY'
from pathlib import Path
base = Path('/path/to/Konglab_data/SCH')
out = base / 'hemispec_api_tutorial_test'
sub = out / 'input_subset'
files = sorted((base / 'GM').glob('*_GM_masked.nii.gz'))[:3]
for p in files:
    target = sub / p.name
    if target.exists() or target.is_symlink():
        target.unlink()
    target.symlink_to(p)
(out / 'selected_subjects.txt').write_text('\n'.join(p.name for p in files) + '\n')
print('\n'.join(p.name for p in files))
PY

    Step 2: run the bilateral workflow through the Python API¶

    The recommended API entry point for a complete two-direction run is run_bilateral_workflow with BilateralWorkflowConfig.

    from pathlib import Path
from hemispec.workflow import BilateralWorkflowConfig, run_bilateral_workflow

base = Path('/path/to/Konglab_data/SCH')
out = base / 'hemispec_api_tutorial_test' / 'api_workflow_out'

config = BilateralWorkflowConfig(
    input_glob=str(base / 'hemispec_api_tutorial_test' / 'input_subset' / '*_GM_masked.nii.gz'),
    out_dir=out,
    device='cpu',  # use 'cuda' or 'auto' on a GPU node when available
    roi_atlas=Path('/path/to/atlas/SCH_1p5mm_atlases/MNI_Glasser_HCP_v1.0_1p5mm.nii.gz'),
    roi_label_table=Path('/path/to/atlas/SCH_1p5mm_atlases/MNI_Glasser_HCP_v1.0_1p5mm_labels.tsv'),
    export_roi_table=True,
    run_classifier=False,
    run_trt=False,
    keep_intermediate=True,  # keep combined_maps for manual-output comparison
    verbose_every=1,
)

result = run_bilateral_workflow(config)
print('RESULT_OUT_DIR', result.out_dir)
print('RESULT_COMBINED_MAPS_DIR', result.combined_maps_dir)
print('RESULT_HEMI_MAPS_DIR', result.hemi_maps_dir)
print('RESULT_ROI_CSV', result.roi_csv)
print('RESULT_ROI_WIDE_CSV', result.roi_wide_csv)
print('RESULT_SUBJECT_SUMMARY_CSV', result.subject_summary_csv)

    Run it on the cluster:

    $PY run_sch_api_workflow.py 2>&1 | tee "$OUT/logs/run_api_workflow.log"

    API calls used¶

    • BilateralWorkflowConfig(...): declares the input glob, output directory, device, atlas, and optional validation flags.
    • run_bilateral_workflow(config): runs both DGN directions, computes ANS/RNS, merges left/right maps, and exports tables.
    • Internally this calls run_pipeline, DGNInferenceConfig, MetricComputeConfig, and the ROI summarizer.

    Equivalent CLI command¶

    The same workflow can be launched with the CLI:

    hemispec workflow \
  --input-glob "$OUT/input_subset/*_GM_masked.nii.gz" \
  --out-dir "$OUT/cli_workflow_out" \
  --device cpu \
  --roi-atlas /path/to/atlas/SCH_1p5mm_atlases/MNI_Glasser_HCP_v1.0_1p5mm.nii.gz \
  --roi-label-table /path/to/atlas/SCH_1p5mm_atlases/MNI_Glasser_HCP_v1.0_1p5mm_labels.tsv \
  --keep-intermediate \
  --verbose-every 1

    Use --device cuda on a GPU node when CUDA is available.

    Step 3: compare API voxel maps against the manual outputs¶

    The manual reference stores whole-brain ANS/RNS maps under ANSRNS_bi_stable/voxelwise/{ANS,RNS}. The API bilateral workflow stores comparable whole-brain maps under api_workflow_out/intermediate/combined_maps when keep_intermediate=True.

    from pathlib import Path
import json
import numpy as np
import pandas as pd
import nibabel as nib

base = Path('/path/to/Konglab_data/SCH')
out = base / 'hemispec_api_tutorial_test'
new_dir = out / 'api_workflow_out' / 'intermediate' / 'combined_maps'
manual_root = base / 'ANSRNS_bi_stable' / 'voxelwise'
subjects = [p.name.replace('_GM_masked.nii.gz', '') for p in sorted((out / 'input_subset').glob('*_GM_masked.nii.gz'))]

rows = []
for subj in subjects:
    for kind in ['ANS', 'RNS']:
        new_path = new_dir / f'{subj}_{kind}.nii.gz'
        manual_path = manual_root / kind / f'{subj}_{kind}.nii.gz'
        new_img = nib.load(str(new_path))
        manual_img = nib.load(str(manual_path))
        new = np.asanyarray(new_img.dataobj).astype(np.float64)
        manual = np.asanyarray(manual_img.dataobj).astype(np.float64)
        finite = np.isfinite(new) & np.isfinite(manual)
        diff = new[finite] - manual[finite]
        absdiff = np.abs(diff)
        corr = np.corrcoef(new[finite].ravel(), manual[finite].ravel())[0, 1]
        rows.append({
            'subject': subj,
            'kind': kind,
            'n_finite_overlap': int(finite.sum()),
            'mean_abs_diff': float(absdiff.mean()),
            'median_abs_diff': float(np.median(absdiff)),
            'max_abs_diff': float(absdiff.max()),
            'rmse': float(np.sqrt(np.mean(diff ** 2))),
            'corr': float(corr),
        })

df = pd.DataFrame(rows)
compare_dir = out / 'comparison'
compare_dir.mkdir(parents=True, exist_ok=True)
df.to_csv(compare_dir / 'api_vs_manual_voxelwise_comparison.csv', index=False)
print(df.to_string(index=False))

    Step 4: compare ROI feature tables¶

    The API writes both long and wide ROI tables:

    api_workflow_out/tables/roi_features_bilateral.csv
api_workflow_out/tables/roi_features_bilateral_wide.csv

    Manual wide-table columns use names such as GLS_ANS_L_roi_1; API wide-table columns use ANS.L_roi_1. Normalize the names before comparing:

    import numpy as np
import pandas as pd
from pathlib import Path

base = Path('/path/to/Konglab_data/SCH')
out = base / 'hemispec_api_tutorial_test'
manual = pd.read_csv(base / 'ANSRNS_bi_stable' / 'ROI_Glasser' / 'SCH_Glasser_ANS_RNS_roi_means.csv')
api = pd.read_csv(out / 'api_workflow_out' / 'tables' / 'roi_features_bilateral_wide.csv')

manual_norm = manual.rename(columns={'subject_id': 'subject'}).copy()
rename = {}
for col in manual_norm.columns:
    if col.startswith('GLS_'):
        parts = col.split('_')
        if len(parts) == 5 and parts[3] == 'roi':
            rename[col] = f'{parts[1]}.{parts[2]}_roi_{parts[4]}'
manual_norm = manual_norm.rename(columns=rename)

subjects = list(api['subject'])
manual_sub = manual_norm[manual_norm['subject'].isin(subjects)].set_index('subject')
api_sub = api.set_index('subject')
common = sorted(set(api_sub.columns).intersection(manual_sub.columns))

rows = []
for subj in subjects:
    for feat in common:
        a = api_sub.loc[subj, feat]
        m = manual_sub.loc[subj, feat]
        if pd.notna(a) and pd.notna(m):
            rows.append({'subject': subj, 'feature': feat, 'api_value': a, 'manual_value': m, 'abs_diff': abs(float(a - m))})
comp = pd.DataFrame(rows)
summary = comp.groupby('subject')['abs_diff'].agg(['count', 'mean', 'median', 'max']).reset_index()
print(summary)

    Full-cohort SCH validation (N=948)¶

    For the formal validation, all available SCH-folder GM maps were processed on the cluster:

    input GM maps:             948
final hemisphere maps:     3,792  # 948 × ANS/RNS × L/R
combined ANS/RNS maps:     1,896  # 948 × ANS/RNS
ROI features per subject:  720    # ANS/RNS × L/R × 180 Glasser ROIs

    The DGN/ANS/RNS workflow was launched as a GPU Slurm job, using device="auto", keep_intermediate=True, Glasser ROI export, and classifier validation. The voxel maps were complete; the original PyPI 0.1.0 ROI export path was slow for full-cohort ROI aggregation, so a vectorized post-processing pass was used to generate the ROI wide table and classifier outputs. The repository implementation has since been optimized to use one-pass label aggregation for mean ROI summaries.

    Hemisphere classification was run from the extracted ROI features. Each metric has 1,896 samples (948 left-hemisphere rows and 948 right-hemisphere rows).

    Summary for presentations:

    • Single-row classifier mean accuracy: (0.8180 + 0.7605) / 2 = 0.7893.
    • Paired-residual classifier mean accuracy: (0.8903 + 0.8660) / 2 = 0.8782.
    • The paired-residual mode is the better full-cohort reporting number because it explicitly removes subject-pair intensity shifts before classifying left vs right ROI profiles.

    Talking points for introducing the API¶

    • The high-level API is compact: configure BilateralWorkflowConfig, then call run_bilateral_workflow.
    • CLI and Python API are aligned; use CLI for batch jobs and Python API for notebooks, custom comparisons, or downstream pipelines.
    • keep_intermediate=True is useful for validation against legacy/manual pipelines; turn it off for production to save storage.
    • Voxelwise and ROI agreement with the manual SCH run is nearly identical at full-cohort scale; residual differences are sparse and consistent with boundary/mask handling.
    • ROI tables are exported in both long and wide forms. The wide form is convenient for classifier/downstream analysis, while the long form is easier to audit.
    • The full-cohort hemisphere-classification analysis is a useful downstream quality check; in this SCH validation, paired-residual accuracy averaged 87.82% across ANS and RNS.
